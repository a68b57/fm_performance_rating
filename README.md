# 园区智驾体验分 - 路书记录页面

这是一个用于产品经理体验园区智驾并给出体验分的路书记录系统。

## 功能说明

### 1. 综合场景评分（权重60%）

包含6个场景，每个场景可以多次记录：

- **进出园区**（权重20%）
- **进出闸机**（权重25%）
- **直角弯（路沿&立柱）**（权重15%）
- **障碍物（VRU&静态）绕行**（权重20%）
- **会车**（权重10%）
- **进出坡道**（权重10%）

每个场景的评分标准：
- 1分：不接肯定不行（-5分/次）
- 2分：不一定要接，但还是有点不放心所以接了（-2分/次）
- 3分：不需要接管，但是开得磕磕绊绊（0分/次）
- 4分：符合预期了，让它开我放心了（+2分/次）
- 5分：完美超预期，比自己开得还好（+5分/次）

每个场景初始分60分，满分100分，最低0分。

### 2. 微行为评分（权重40%）

初始分100分，出现一次扣3分：

- 顿挫
- 不居中
- 无故低速
- 速度偏快
- 反复修正方向盘
- 异常降级/退出

### 3. 用户bonus（权重100%）

初始分0分，最高20分，最低0分：

- 好场景用户主动点赞：+3分/次
- Base case 用户额外惩罚：-3分/次

## 分数计算

**总分 = 0.6 × 综合场景分 + 0.4 × 微行为得分 + 用户bonus**

其中：
- 综合场景分 = 各子场景得分 × 子场景权重求和
- 微行为得分 = 100 - (各微行为次数 × 3)
- 用户bonus = 点赞次数 × 3 - 惩罚次数 × 3（最高20，最低0）

## 使用方法

1. 打开 `index.html` 文件
2. 在体验过程中，遇到场景时点击对应场景的"添加记录"按钮
3. 选择该场景的评分（1-5分）
4. 记录微行为出现的次数
5. 记录用户bonus（点赞或惩罚）
6. 系统会自动计算并显示最终得分
